# Knowledge Graph Module --- Overview (Obsidian / Roam Research style)

Updated: 2026-03-04

The Knowledge Graph module is designed to act as a **Kortex
system** where information is stored as interconnected notes.

The goal is to create a **network of knowledge**, where ideas, notes,
concepts, and information connect together.

------------------------------------------------------------------------

## Core Principles

This module follows the principles of:

-   Networked thinking
-   Bidirectional linking
-   Knowledge graph visualization
-   Atomic notes
-   Personal knowledge management (PKM)

------------------------------------------------------------------------

## Core Concepts

### Node (Note)

A node represents a piece of knowledge.

Examples:

-   Idea
-   Concept
-   Research note
-   Journal entry
-   Book note

Each node may contain:

Title\
Content\
Tags\
Links\
Metadata

------------------------------------------------------------------------

### Links

Nodes can link to other nodes.

Example:

Note A → Note B

Links automatically generate **backlinks**.

------------------------------------------------------------------------

### Backlinks

If Note A links to Note B, Note B shows:

Referenced by → Note A

This creates a **bidirectional knowledge graph**.

------------------------------------------------------------------------

### Graph

The system visualizes notes as a graph:

Nodes = Notes\
Edges = Links

------------------------------------------------------------------------

### Blocks

Notes may contain **blocks**.

Blocks allow:

-   granular linking
-   block references
-   block embedding

Example:

Paragraph → referenceable block

------------------------------------------------------------------------

### Tags

Tags classify knowledge.

Examples:

#philosophy\
#startup\
#productivity

------------------------------------------------------------------------
