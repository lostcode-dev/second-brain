O Kortex usa **Nuxt Content**, então é possível enriquecer páginas com componentes.

## Quando usar

- Guias internos (ex.: onboarding)
- Páginas com “próximos passos”
- Conteúdo que se beneficia de layout (cards, alertas)

## Regra de ouro

Não transforme documentação em UI complexa. Priorize legibilidade.

Se precisar de elementos como “dica”, “aviso” e “passo a passo”, use componentes já existentes do projeto (Nuxt UI / Nuxt Content).
